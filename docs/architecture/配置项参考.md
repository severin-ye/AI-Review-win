# 配置项参考

> 代码位置：`backend/app/services/user_settings.py`（DEFAULT_SETTINGS、读取与截断）、`backend/app/api/settings.py`（REST 接口与 api_key 掩码）、`backend/app/config.py`（pydantic Settings 与环境变量）。
> 本文是全部用户设置键、环境变量与示例配置的权威清单。

## 1. 用户设置（settings 表，REST 读写）

`GET /api/settings` 返回合并后的完整配置；`PUT /api/settings` 接受部分键更新并做合法性截断。默认值（DEFAULT_SETTINGS）共 16 键：

### 1.1 LLM

| 键 | 默认 | 说明 |
|---|---|---|
| `llm.base_url` | `""` | OpenAI 兼容接口地址。示例：DeepSeek `https://api.deepseek.com`；OpenAI 官方 `https://api.openai.com/v1`（设置页 placeholder） |
| `llm.api_key` | `""` | 明文存库，API 出参掩码（见 §4） |
| `llm.model` | `""` | 模型名，如 `deepseek-chat` |
| `review.prompt` | 内置模板 | 审校 system prompt，要点：扮演资深中文编辑；逐条输出 {original, suggestion, reason, error_type, severity} 的 JSON 数组；error_type ∈ 错别字/语法/标点/术语/风格/事实核查；severity ∈ error/warning/info；只报有把握的问题 |

### 1.2 检索（retrieve）

| 键 | 默认 | 合法范围（越界自动截断） |
|---|---|---|
| `retrieve.query_count` | `8` | 5 – 10 |
| `retrieve.vector_topk` | `3` | 1 – 20 |
| `retrieve.bm25_topk` | `3` | 1 – 20 |
| `retrieve.rrf_k` | `60` | 1 – 1000 |
| `retrieve.enabled` | `true` | 布尔；为 false 时调 `/api/retrieve` 返回 400 |

### 1.3 Embedding

| 键 | 默认 | 说明 |
|---|---|---|
| `embedding.provider` | `"local"` | ∈ `local` / `openai` / `stub`（stub 供测试） |
| `embedding.model` | `"BAAI/bge-m3"` | 本地模型名/路径 |

### 1.4 分段（segment）

| 键 | 默认 | 合法范围 |
|---|---|---|
| `segment.min_sentence_length` | `10` | ≥ 1（短于该长度的句与相邻句合并） |
| `segment.review_references` | `false` | 是否审校参考文献段 |

### 1.5 导出（docx / output）

| 键 | 默认 | 合法范围 |
|---|---|---|
| `docx.has_review_table` | `"Y"` | 原稿是否含审校表格（Y/N 系布尔，见 §3） |
| `docx.first_line_indent` | `0.5` | ≥ 0，首行缩进（字符） |
| `output.dir` | `""` | 自定义导出目录；空 = `<data_dir>/projects/<doc_id>/exports/` |

### 1.6 键名兼容与类型转换

- **点分键与下划线旧写法双向兼容**：`retrieve.rrf_k` 与 `retrieve_rrf_k` 都会归一到点分键存储。
- 布尔接受 `"Y"/"YES"/"TRUE"/"1"`（大小写不敏感）为真。
- 数值非法（非数字、越界）时回落默认值，不会写入脏值。

## 2. 环境变量

### 2.1 后端读取

| 变量 | 作用 | 读取位置 |
|---|---|---|
| `AI_REVIEW_DATA_DIR` | 数据目录最高优先级覆盖（测试隔离靠它） | `config.py` |
| `AI_REVIEW_PACKAGED` | `=1` 时数据目录切到 `%APPDATA%/ai-review`；**由 Electron sidecar 生产模式注入，是后端唯一读取的注入变量** | `config.py` |
| `AI_REVIEW_SAT_MODEL_PATH` / `AI_REVIEW_SAT_TOKENIZER_PATH` | 分段模型/分词器路径覆盖 | `model_manager.py` |
| `AI_REVIEW_EMBEDDING_MODEL_PATH` | embedding 模型路径覆盖 | `model_manager.py` |
| `AI_REVIEW_SEGMENTER` | `=rule` 强制规则分段器（跳过 SAT 模型，测试用） | segmenter 选择处 |
| `HF_ENDPOINT` | HuggingFace Hub 镜像（如 `https://hf-mirror.com`） | transformers/hub 标准变量 |
| `PORT` | 后端监听端口（默认 8765） | `run.py`（直接跑 Python 时） |
| `AI_REVIEW_` 前缀（pydantic） | `Settings(env_prefix="AI_REVIEW_")`：`app_name` / `version` / `data_dir` 可被同名环境变量覆盖 | `config.py` |

### 2.2 sidecar 注入但后端不读取 ⚠️

| 变量 | 实际情况 |
|---|---|
| `AI_REVIEW_DEV` | Electron sidecar 在开发模式下注入，但 grep 证实**后端代码从未读取它**。开发/生产的判别实际只依赖 `AI_REVIEW_PACKAGED`。如实记录，属冗余注入。 |

### 2.3 前端 / 桌面端

| 变量 | 作用 | 读取位置 |
|---|---|---|
| `ELECTRON_RENDERER_URL` | dev 时 Electron 加载该 URL（Vite dev server）而非 file:// | `electron/main.cjs` |
| `ELECTRON_MIRROR` | `.npmrc` 中配置 `https://npmmirror.com/mirrors/electron/`，加速 Electron 二进制下载；安装时若报镜像警告属无害 | npm/pnpm |

## 3. pydantic Settings（应用级）

`backend/app/config.py` 的 `Settings` 只有三个字段，均可用 `AI_REVIEW_` 前缀环境变量覆盖：

| 字段 | 默认 | 说明 |
|---|---|---|
| `app_name` | `"ai-review"` | |
| `version` | `"0.1.0"` | `/api/health` 返回 |
| `data_dir` | 按 §2.1 优先级解析 | |

## 4. api_key 掩码与安全

- `GET /api/settings`：`llm.api_key` 以 `****` + 后 4 位返回（长度 ≤ 4 时全掩码）。
- `PUT /api/settings`：收到的值以 `****` 开头视为「未修改」，跳过写入；收到**空字符串**表示明确清除。
- 库内为明文（本机 SQLite）；keytar 系统钥匙串存储在设计文档中列为后续增强，当前未实现。

## 5. 典型配置示例

DeepSeek：

```json
{
  "llm.base_url": "https://api.deepseek.com",
  "llm.api_key":  "sk-xxxxxxxx",
  "llm.model":    "deepseek-chat"
}
```

关闭检索、纯 LLM 审校（无 KB 时的降级用法）：

```json
{ "retrieve.enabled": false }
```

此时调 `/api/sentences/{id}/retrieve` 会返回 400，但 `/api/review` 仍可用（审校不强制依赖证据）。
