# 前端详解

> 代码位置：`frontend/src/`（React 19 + Vite 8）。本文逐页逐模块说明前端结构、状态管理、API 客户端与样式体系。
> 设计出处：[`../design/前端设计文档.md`](../design/前端设计文档.md)、[`../design/交互设计文档.md`](../design/交互设计文档.md)。

## 1. 技术要点

| 项 | 实现 |
|---|---|
| 框架 | React 19.2 + TypeScript 5.9（strict） |
| 构建 | Vite 8 + `@vitejs/plugin-react` |
| 路由 | react-router-dom 7.13 **HashRouter** |
| 服务端状态 | `@tanstack/react-query` 5（健康检查等） |
| 客户端状态 | zustand 5（workbenchStore / appStore） |
| 样式 | Tailwind CSS v4（`@tailwindcss/vite` 插件，无配置文件） |
| 组件 | 手工 shadcn 风格组件 6 个（无 radix 依赖）+ lucide-react 图标 |
| 包管理 | pnpm 10（lockfileVersion 9；npm 亦可） |
| Node 要求 | `>=20.19.0`（Vite 8 要求 20.19+ 或 22.12+） |

用 HashRouter 而不用 BrowserRouter 的原因：生产环境下渲染进程从 `file://` 加载 `dist/index.html`，History API 路由在 `file://` 协议下不可用；Hash 路由对 file 协议和静态服务器都天然兼容。

## 2. 入口与路由

`main.tsx`：`StrictMode` + `QueryClientProvider`（`new QueryClient()` 默认配置）挂载 `<App/>`。

`App.tsx` 路由表：

| 路径 | 页面 | 说明 |
|---|---|---|
| `/` | → Navigate `/documents` | 默认页 |
| `/documents` | DocumentsPage | 文档库 |
| `/workbench/:docId?` | WorkbenchPage | 审校工作台，docId 可选 |
| `/kb` | KnowledgeBasePage | 知识库管理 |
| `/settings` | SettingsPage | 系统设置 |

所有页面共用 `MainLayout`：左侧 Sidebar（品牌区"AI 审校助手"+ 三个导航项 + 底部设置入口），右侧内容区顶部是 `BackendStatus` 横幅，再下面是页面本体（`max-w-6xl` 居中，唯独 WorkbenchPage 用 `max-w-none` 全宽）。

`BackendStatus.tsx`（106 行）：react-query 每 5 秒轮询 `GET /api/health`（`retry: false`），三态显示——绿色点"后端服务正常"、琥珀色脉冲点"正在连接后端服务…"、红色点"后端服务异常"（此时附带 backend 启动命令提示）。页面加载时短暂显示"正在连接"是正常现象。

## 3. API 客户端 `api/client.ts`（499 行）

### 3.1 baseUrl 解析与浏览器预览降级

```text
baseUrlPromise（模块级缓存，只解析一次）
 ├─ window.api?.getBackendUrl?.()  → Electron preload 注入的 http://127.0.0.1:<动态端口>
 └─ 否则回退 http://127.0.0.1:8765  → isBrowserPreview = true
```

`window.api` 的类型声明在 `api/electron.d.ts`（`getBackendUrl()` 与 `getAppInfo()`）。浏览器预览模式下所有写操作/上传会在后端不可达时自然失败；`isBrowserPreview` 标记导出供 UI 降级使用（M4 §5.4）。

`apiFetch` 统一封装：拼接 URL、默认 JSON 头、解析错误体（fastapi 的 `detail` 字段），非 2xx 抛出带状态码的 `Error`。

### 3.2 函数清单（按里程碑分组）

| 分组 | 函数 |
|---|---|
| 类型 | `DocumentRecord / BlockNode / SentenceRecord / CorrectionItem / ReviewResults / KbDocument / ModelStatus / HealthResponse / AppSettings` 等 |
| M2 | `listDocuments / uploadDocument / deleteDocument / getDocumentParsedMd / reparseDocument / getDocumentReview`；`listKbDocuments / uploadKbDocuments / deleteKbDocument / reindexKb` |
| M2 模型管理 | `getModelStatus / downloadModel / cancelDownload / subscribeDownloadEvents` |
| M3 | `getBlockTree / getDocumentSentences / retrieveSentence / updateRetrievalSettings / updateReviewSettings` |
| M4 | `startReview / subscribeReviewEvents` |
| M5 | `decideCorrection / batchDecide / exportDocument` |
| 健康检查 | `fetchHealth` |

### 3.3 SSE 订阅 `subscribeJobEvents`

对 `GET /api/jobs/{job_id}/events` 的原生 `EventSource` 封装，监听后端 pushJobEvent 发出的全部事件类型：

```typescript
['start','loaded','chunked','embedding','bm25','progress','skipped','warning','stage_done','error']
```

外加 `done`（后端关闭连接时 EventSource 触发 error 事件，客户端据 `readyState === CLOSED` 判定正常结束）。返回关闭函数供组件卸载时调用。`startReview` / `downloadModel` / `reindexKb` 返回的 `job_id` 都通过它跟踪。

## 4. 状态管理（zustand）

### 4.1 `workbenchStore.ts`（220 行）——审校工作台核心

- **数据形态**：`FlatCorrection`——把 `GET /documents/{id}/review` 的树形结果（blocks → sentences → corrections）拍平成带 block/sentence 上下文的扁平列表，便于 j/k 顺序导航与筛选。
- `load(docId)`：并行拉 blockTree + sentences + review，组装 FlatCorrection 列表。
- `decide(id, decision, editedText?)`：**乐观更新**——先改本地状态并把条目移出 pending，再发请求；失败时回滚并 `toast.error`（M4 §5.3.1）。decide 成功后自动把 `activeId` 推进到下一个 pending 条目（快捷键流的关键）。
- `batch(decision, onlyVisible)`：批量处理；完成后整体重新 `load`（批量改判可能改变筛选结果，整载最稳）。
- 筛选状态：`severityFilter / typeFilter / sourceFilter` 三件套，`visibleCorrections` 派生。

### 4.2 `appStore.ts`（30 行）

全局 UI 状态：后端连接标记等。**注意：`currentDocumentId` 字段已定义但当前全项目无任何引用（grep 验证），属于预留字段，"审校完成跳转工作台"实际通过路由跳转完成。**

## 5. 页面详解

### 5.1 DocumentsPage（807 行）——文档库

- 文档列表 + 大按钮上传（`.docx`，`uploadDocument`）。
- **状态芯片 `STATUS_META`**：为 10 种状态定义图标/颜色/文案：pending / parsed / segmenting / segmented / retrieving / retrieved / reviewing / done / partial / error。`failed` 映射到 `error`（M4 §6 勘误）。
- **ProgressRing**：SVG 圆环，reviewing 时按 `sentenceStats.done/total` 显示进度百分比（由 2 秒轮询驱动，不依赖 SSE——SSE 只负责完成时刻的跳转）。
- **按状态显示操作按钮**：解析（parsed）/ 检索（segmented）/ AI 审校（retrieved）/ 重试（error）/ 继续审校（partial）/ 查看审校（done）/ 导出成稿 / 删除。
- **行展开**：BlockTree 缩略 + 逐句 evidence 卡片（编号徽章 [n]、来源、chunk 文本）+ parsed.md 预览 + `ExportPanel`（导出成稿/导出标注稿/导出审校对照，显示已导出文件的时间戳）。
- review SSE：只在**完成**时 `navigate(`/workbench/${id}`)`；中途进度靠轮询。
- reviewing / retrieving 文档存在时启动 2 秒 `setInterval` 轮询列表。

### 5.2 WorkbenchPage（624 行）——三栏审校工作台

布局：左栏原文（block 树渲染，当前句高亮）｜中栏修改建议卡（当前 correction 的 diff）｜右栏待处理列表（可点选）。顶栏：进度 x/y、严重度/类型/来源三个筛选、批量接受/拒绝按钮。

四套 META 常量：

- `SEVERITY_META`：error 错误 / warning 建议 / info 提示
- `ERROR_TYPE_META`：错别字 / 语法 / 标点 / 术语 / 风格 / 事实核查
- `DECISION_META`：pending / accepted / rejected / edited
- `SOURCE_META`：llm / retrieval

**SentenceDiff**：用 `indexOf(original)` 在原文中定位被改片段；定位失败时降级为整句片段级 diff（原文红底删除线 / 修订绿底下划线，M4 §5.3.2）。

**快捷键**（输入框聚焦时自动失效）：

| 键 | 动作 |
|---|---|
| `j` / `↓` | 下一条 pending |
| `k` / `↑` | 上一条 pending |
| `1` | 接受当前条 |
| `2` | 拒绝当前条 |
| `3` | 编辑当前条（进入编辑态） |

「拒绝全部」打开确认 Dialog（`components/ui/dialog`，基于原生 `<dialog>` 元素）。

### 5.3 KnowledgeBasePage（259 行）

- KB 文档表格：文件名、chunk 数、状态、上传时间；操作：重建索引（reindex）、删除；indexing 状态禁用操作按钮。
- 多文件上传（`.md/.txt/.docx`）。
- **trackJob**：reindex 返回 job_id 后 `subscribeJobEvents`，把 SSE 事件追加为行日志显示（最多 5 个面板、每面板 20 行，M4 §5.3.3 调整）。

### 5.4 SettingsPage（397 行）

四个分区，字段由 `fields` 配置数组驱动渲染：

1. **LLM**：base_url / api_key（password 输入）/ model / prompt（textarea）
2. **检索**：query_count / vector_topk / bm25_topk / rrf_k / enabled
3. **分段与导出**：segment.min_sentence_length / docx.has_review_table / docx.first_line_indent / output.dir
4. **模型管理**：`ModelManager` 组件——sat-3l-sm / bge-m3 / xlm-roberta 三模型的状态（已下载/未下载/下载中）、大小、下载按钮与 SSE 进度条

保存后立即自动 `testLlmConnection`，连接状态用三态 tag 展示（未测试/成功/失败）；任何 `llm.*` 字段改动会把连接状态重置回 idle（M4 §6 记录）。

## 6. 样式与 UI 组件

- **Tailwind v4**：`index.css` 全文只有 `@import "tailwindcss";` 加少量全局样式——v4 不再需要 `tailwind.config.js`，工具类由 vite 插件按需生成。
- `cn()`（`lib/utils.ts`）= `twMerge(clsx(...))`，条件类名合并工具。
- `components/ui/` 六个手工组件（无 radix-ui 依赖）：
  - `button`（variant/size 用 cva）
  - `badge`
  - `card`（Card/CardHeader/CardTitle/CardDescription/CardContent/CardFooter）
  - `separator`
  - `dialog`（原生 `<dialog>` + `showModal()`）
  - `sonner`（命令式 `toast.success/error/info`，3200ms 自动消失，固定右上角）

## 7. 构建与类型检查

```bash
pnpm build        # = tsc -b && vite build → frontend/dist/
pnpm lint         # eslint 9 flat config
```

`tsconfig.json` 是 references 容器（app/node 两个子配置）；**不含 `baseUrl`**——M4 曾遇到 TS5102「`baseUrl` 不再支持」报错（TS 5.9 行为变化），修复方式是直接删除该字段而不是保留空值。
