# 架构总览

> 范围：进程拓扑、技术栈、端到端数据流、开发/生产模式差异、浏览器预览降级。
> 上游文档：`docs/design/AI-Review-Electron-重构设计.md`（§2 总体架构）；实现细节见同目录《后端详解》《前端详解》《数据模型与持久化》。

## 1. 进程拓扑

应用由 **Electron 三进程 + Python FastAPI sidecar 子进程** 组成。sidecar 不是独立服务，而是由 Electron 主进程按需拉起、随应用退出一并销毁的子进程。

```
┌──────────────────────────── Electron 应用 ────────────────────────────┐
│                                                                      │
│  ┌ 主进程 (Node.js)  electron/main/index.ts + sidecar.ts ─────────┐  │
│  │  1. findFreePort() 向 OS 申请 127.0.0.1 空闲端口（非固定端口）  │  │
│  │  2. spawn Python sidecar（dev: uvicorn / prod: 单文件 exe）     │  │
│  │  3. waitForHealthy() 轮询 GET /api/health（300ms×至多60s）      │  │
│  │  4. 就绪后创建 BrowserWindow（1440×900，contextIsolation）      │  │
│  │  IPC handlers: backend:url / app:version / shell:showItemInFolder│ │
│  └──────┬───────────────────────────────────────▲────────────────┘  │
│         │ spawn/stdout                            │ ipcRenderer.invoke│
│         ▼                                         │                 │
│  ┌ Python FastAPI sidecar ────────────┐   ┌ preload ──────────────┐ │
│  │  uvicorn app.main:app               │   │ contextBridge 暴露     │ │
│  │  监听 http://127.0.0.1:<动态端口>   │   │ window.api = {         │ │
│  │  业务全部走 HTTP REST + SSE         │   │   getBackendUrl()      │ │
│  │  （见《后端详解》）                  │   │   getAppVersion()      │ │
│  └─────────────────────────────────────┘   │   showItemInFolder() }│ │
│                                             └──────────┬───────────┘ │
│                                                        │ window.api  │
│                                             ┌ Renderer (React SPA) ─┤
│                                             │ HashRouter + 四页面    │
│                                             │ fetch(baseUrl+/api/…) │
│                                             │ EventSource 订阅 SSE   │
│                                             └───────────────────────┘
└──────────────────────────────────────────────────────────────────────┘
```

### 1.1 端口约定（重要）

- **Electron 模式（dev 与 prod 一致）：端口由 `findFreePort()` 动态分配**（`electron/main/sidecar.ts`，`server.listen(0, '127.0.0.1')`），避免与既有服务冲突。渲染层启动后经 `window.api.getBackendUrl()`（IPC `backend:url`）拿到形如 `http://127.0.0.1:54321` 的 baseUrl。
- **`npm run backend:dev` 手动调试后端：固定 127.0.0.1:8765**（package.json 脚本写死 `--port 8765`），浏览器预览模式的前端回退地址也是它。
- `backend/run.py`（PyInstaller 入口）的默认回退端口同样是 8765（`--port` 参数或 `PORT` 环境变量优先）；prod 下 Electron 总会显式传 `--port <动态端口>`。

### 1.2 sidecar 的拉起与守护（`electron/main/sidecar.ts`）

| 环节 | dev（`app.isPackaged=false`） | prod（打包后） |
|---|---|---|
| 后端目录 | `<仓库>/backend/` | `process.resourcesPath/backend/`（electron-builder `extraResources` 映射） |
| 命令 | `backend/.venv/Scripts/python.exe -m uvicorn app.main:app --port <port> --host 127.0.0.1 --reload`（venv 不存在则回退系统 `python`） | `ai-review-backend.exe --port <port>` |
| 注入环境变量 | `AI_REVIEW_DEV=1`、`AI_REVIEW_PACKAGED=0` | `AI_REVIEW_DEV=0`、`AI_REVIEW_PACKAGED=1`（数据目录因此切到 `%APPDATA%/ai-review/`，见《数据模型与持久化》） |
| 日志 | stdout/stderr 逐行经 `onLog` 透传到主进程控制台（`[sidecar]` / `[sidecar:err]` 前缀） | 同左（spec 中 `console=True` 保留控制台） |
| 就绪判定 | `waitForHealthy()`：每 300ms GET `/api/health`，200 即返回；60s 超时抛错 | 同左 |
| 退出清理 | `before-quit` → `stopSidecar()`：Windows 用 `taskkill /pid <pid> /T /F` 杀整棵进程树（覆盖 `--reload` 派生的子进程），其它平台 `SIGTERM` | 同左 |

主进程无 sidecar 崩溃自动重启逻辑：sidecar 退出仅记日志（`[sidecar] exited, code=...`），窗口仍在，前端「后端未连接」红灯（`BackendStatus` 5s 轮询）可见。

### 1.3 window.api 桥（`electron/preload/index.ts`）

预加载脚本在 `contextIsolation: true` 下仅暴露三个方法（无 Node 能力泄漏）：

| 方法 | IPC 通道 | 用途 |
|---|---|---|
| `getBackendUrl(): Promise<string>` | `backend:url` | 取 sidecar baseUrl（渲染层唯一获取后端地址的途径） |
| `getAppVersion(): Promise<string>` | `app:version` | 应用版本号（`app.getVersion()`，来自 package.json） |
| `showItemInFolder(path): Promise<void>` | `shell:showItemInFolder` | 在系统文件管理器中定位导出产物（主进程仅接受非空字符串，由渲染层传入后端返回的绝对路径） |

类型声明在 `renderer/src/api/electron.d.ts`（`Window.api` 全局扩展）。

## 3. 端到端数据流

以一篇文稿从上传到导出双版本的完整时序（同步/异步标注见注释）：

```
用户                Electron 渲染层                FastAPI sidecar                     落盘/落库
 │                       │                              │                                  │
 │ 上传 docx             │ POST /api/documents          │ 写 projects/<doc_id>/original.docx │
 │──────────────────────>│─────────────────────────────>│ documents 行 status=uploaded      │
 │                       │                              │                                  │
 │ 点「解析」            │ POST /documents/{id}/run     │【同步】job(type=pipeline)          │
 │──────────────────────>│─────────────────────────────>│ ①ingest: 移除首个表格(可配)→抽表   │
 │                       │                              │   →tables.docx；占位符→no_table.docx│
 │                       │                              │   →parsed.md；status: uploaded→parsed│
 │                       │                              │ ②segment: SaT 分句→短句合并→章节/  │
 │                       │                              │   参考文献分块→blocks/sentences 入库│
 │                       │<── 返回 blocks/sentences 数 ──│   status: parsed→segmented         │
 │                       │                              │                                  │
 │ 知识库页上传参考文档   │ POST /api/kb/documents       │ 写 kb/files/<kb_id>.<ext>          │
 │──────────────────────>│─────────────────────────────>│【后台线程】job(type=kb_index)：     │
 │                       │ 返回 job_id                  │ 加载→切块(500/50)→BGE-M3 嵌入→     │
 │                       │                              │ LanceDB upsert→重建 BM25(bm25.pkl) │
 │ ◄── SSE 进度 ─────────│ EventSource                  │ kb_documents: indexing→indexed     │
 │                       │  /api/jobs/{job_id}/events   │                                  │
 │                       │                              │                                  │
 │ 点「检索」            │ POST /documents/{id}/retrieve│【同步】job(type=retrieve)：每句     │
 │──────────────────────>│─────────────────────────────>│ 查询重写(LLM, 默认8问, 失败降级原句)│
 │                       │                              │ →向量路 top3 + BM25路 top3(RRF k=60)│
 │                       │                              │ →去重≤6 条→queries/evidence 入库    │
 │                       │<── 返回 句数/证据数 ──────────│ status: retrieving→retrieved       │
 │                       │                              │                                  │
 │ 点「AI 审校」         │ POST /documents/{id}/review  │ LLM 配置预检(缺→400 不动数据)       │
 │──────────────────────>│─────────────────────────────>│【后台线程】job(type=review)：       │
 │                       │ 返回 job_id                  │ 清 pending corrections(默认)        │
 │ ◄── SSE: start/       │ EventSource                  │ →status=reviewing→逐块送审(chat_json)│
 │     progress/warning  │  /api/jobs/{job_id}/events   │ →解析校验→corrections 入库(pending) │
 │     /done             │                              │ →无 pending→manual_done 否则        │
 │                       │                              │   pending_manual；失败→failed       │
 │ 完成自动跳工作台       │                              │                                  │
 │                       │                              │                                  │
 │ 逐条决定(1/2/3)        │ POST /corrections/{id}/decision│ 写 decision/decided_at/custom_text│
 │──────────────────────>│─────────────────────────────>│ 全部决定→manual_done；撤销→pending_manual│
 │ 批量接受/拒绝          │ POST /documents/{id}/decisions/batch │ 同上收口                  │
 │                       │                              │                                  │
 │ 点「导出成稿」        │ POST /documents/{id}/export  │【同步】job(type=export)：按决定重建  │
 │──────────────────────>│─────────────────────────────>│ 段落→清洁版(_审校修订1_)/留痕版     │
 │                       │                              │ (_审校修订2_)→表格还原→首行缩进     │
 │                       │<── 返回 路径/adopted/warnings│ documents.exports_json；status→done │
 │ 打开所在文件夹/下载    │ IPC showItemInFolder 或      │ exports/ 目录产物                   │
 │                       │ GET /documents/{id}/exports/{kind} │                              │
```

要点：

- **同步接口**：run（解析+分句）、retrieve、export——HTTP 请求阻塞到完成，job/job_events 仅作留痕与事后回放。
- **后台线程接口**：review、kb 上传/reindex、models/download——立即返回 `job_id`，前端用 `EventSource` 订阅 `/api/jobs/{job_id}/events` 拿进度（SSE 从 `job_events` 表回放，见《后端详解》§4）。
- 状态机的完整定义与转移条件见《后端详解》§3。

## 4. 开发模式 vs 生产模式

| 维度 | dev（`npm run dev`） | prod（安装包） |
|---|---|---|
| 渲染层加载 | `ELECTRON_RENDERER_URL`（electron-vite dev server，HMR） | `loadFile(out/renderer/index.html)`（HashRouter 保证 file:// 下路由可用） |
| 后端进程 | `backend/.venv` uvicorn `--reload`（源码改动自动重载） | `resources/backend/ai-review-backend.exe`（PyInstaller onefile，`run.py` 入口，`multiprocessing.freeze_support()`） |
| 环境变量 | `AI_REVIEW_DEV=1` / `AI_REVIEW_PACKAGED=0` | `AI_REVIEW_DEV=0` / `AI_REVIEW_PACKAGED=1` |
| 数据目录 | `backend/.data/`（可用 `AI_REVIEW_DATA_DIR` 覆盖） | `%APPDATA%/ai-review/` |
| 模型目录 | `backend/.data/models/`（自动探测） | `%APPDATA%/ai-review/models/`（自动探测；安装包不含模型，设置页预下载） |
| 端口 | 动态空闲端口（`backend:dev` 手动模式才用 8765） | 动态空闲端口 |
| 日志 | sidecar stdout 透传主进程控制台 | 同左（exe 保留 console） |

打包链路（详见《开发指南》§5）：`npm run dist:backend` 出 `backend/dist/ai-review-backend.exe` → `electron-builder.yml` 的 `extraResources` 映射到 `resources/backend/` → `npm run dist` 出 NSIS 安装包（`release/`，非一键安装、可改目录、桌面+开始菜单快捷方式，productName「AI 审校助手」，appId `com.ai-review.app`）。

## 5. 浏览器预览降级机制

目的：不启动 Electron 也能调试 UI（`npm run dev:renderer` + `npm run backend:dev`）。

- **判定**：`renderer/src/api/client.ts` 的 `isBrowserPreview()` —— `window.api` 不存在即为浏览器预览。
- **baseUrl 回退**：`getBaseUrl()` 在无 `window.api` 时固定返回 `http://127.0.0.1:8765`（配合 `backend:dev`）；该 Promise 只解析一次并缓存。
- **UI 提示**：`App.tsx` 顶部显示琥珀色提示条「浏览器预览模式：后端连接 http://127.0.0.1:8765（npm run backend:dev）；『打开所在文件夹』等系统集成功能不可用，导出产物以下载链接代替」。
- **功能降级**：仅「打开所在文件夹」（依赖 IPC）不可用，导出产物面板改为 `<a download>` 链接（`GET /api/documents/{id}/exports/{kind}`，FileResponse 直出 docx）。其余业务 API（含 SSE）在浏览器下完全可用——后端 CORS 放行 `http://(localhost|127.0.0.1)(:端口)?` 来源。
- 独立 vite 配置 `renderer/vite.config.ts`（react + tailwind + `@` alias，`host: 127.0.0.1`，`strictPort: false`）与 electron-vite 的 renderer 配置等价，页面产物一致。
