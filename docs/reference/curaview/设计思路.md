# GraphRAG幻觉检测Agent - 详细设计思路文档

## 一、项目概述

### 1.1 项目目标
构建一个基于LangChain 1.0和GraphRAG的智能幻觉检测Agent,能够:
- 对医学文本按句子进行幻觉检测
- 利用GraphRAG的知识图谱进行证据检索
- 输出结构化的幻觉检测报告

### 1.2 核心创新点
1. **知识图谱驱动**: 使用GraphRAG的实体、关系、社区报告作为事实依据
2. **句子级检测**: 精细化到句子粒度的幻觉分析
3. **多维度评估**: 包含幻觉类型、证据等级、推理过程的完整输出
4. **LangChain 1.0 Agent架构**: 采用最新的`create_agent`统一入口

### 3.3 幻觉检测提示词工程

#### 提示词设计原则
1. **角色定位**: "你是专业的医学幻觉检测专家"
2. **任务明确**: "检测以下句子是否包含医学幻觉"
3. **证据导向**: "基于以下GraphRAG检索的证据进行判断"
4. **类型枚举**: 明确列出7种幻觉类型
5. **等级定义**: 清晰定义E1-E4证据等级

#### 提示词模板
```
# 系统提示词
你是一名专业的医学幻觉检测专家。你的任务是检测医学文本中的幻觉(与事实不符的陈述)。

## 幻觉类型定义
1. diagnosis_error: 诊断错误
2. medication_error: 用药错误
3. exam_result_error: 检查结果错误
4. time_error: 时间错误
5. value_error: 数值错误
6. negation_error: 否定错误
7. invented_fact: 无中生有

## 证据等级定义
- E1: 结构化字段支持 (有明确的数据库记录)
- E2: 非结构化证据支持 (在文本中有描述)
- E3: 无证据支持 (既无结构化也无非结构化证据)
- E4: 与已有证据冲突 (直接矛盾)

## 检测流程
1. 阅读待检测句子
2. 查看GraphRAG提供的证据
3. 逐一比对事实
4. 给出判断和理由

# 用户提示词
待检测句子: "{sentence}"

GraphRAG检索到的相关证据:
{graphrag_context}

请输出你的检测结果。
```

### 3.4 结构化输出Schema

#### Pydantic模型定义
```python
from pydantic import BaseModel, Field
from enum import Enum

class HallucinationType(str, Enum):
    """幻觉类型"""
    DIAGNOSIS_ERROR = "diagnosis_error"
    MEDICATION_ERROR = "medication_error"
    EXAM_RESULT_ERROR = "exam_result_error"
    TIME_ERROR = "time_error"
    VALUE_ERROR = "value_error"
    NEGATION_ERROR = "negation_error"
    INVENTED_FACT = "invented_fact"

class EvidenceGrade(str, Enum):
    """证据等级"""
    E1 = "E1"  # 结构化字段支持
    E2 = "E2"  # 非结构化证据支持
    E3 = "E3"  # 无证据支持
    E4 = "E4"  # 与已有证据冲突

class HallucinationDetectionResult(BaseModel):
    """幻觉检测结果 - AI输出部分"""
    reasoning: str = Field(..., description="有无幻觉的原因和思考过程")
    hallucination_status: bool = Field(..., description="是否存在幻觉,True表示存在")
    hallucination_type: Optional[HallucinationType] = Field(None, description="幻觉类型")
    evidence_grade: EvidenceGrade = Field(..., description="证据等级")

class SentenceDetectionRecord(BaseModel):
    """单个句子的完整检测记录 - 最终拼接版"""
    sentence_index: int = Field(..., description="句子编号")
    original_sentence: str = Field(..., description="原始句子")
    detection_result: HallucinationDetectionResult = Field(..., description="AI检测结果")
    graphrag_evidence: str = Field(..., description="GraphRAG提供的原始证据")
```

## 六、关键技术决策

### 6.1 为什么使用LangChain 1.0 Agent?
1. **统一入口**: `create_agent`简化代码
2. **工具编排**: 自动处理工具调用的ReAct循环
3. **记忆管理**: `checkpointer`支持多轮对话
4. **可观测性**: 天然集成LangSmith追踪

### 6.2 为什么将GraphRAG封装为Tool?
1. **解耦**: Agent不需要知道GraphRAG的内部实现
2. **可替换**: 未来可以替换为其他知识库
3. **LangChain生态**: 符合LangChain的工具调用范式

### 6.3 为什么采用句子级检测?
1. **精确定位**: 可以准确指出哪一句有问题
2. **证据对应**: GraphRAG可以针对具体句子检索证据
3. **人类可读**: 方便医生审核和修正

## 七、潜在挑战与解决方案

### 7.1 性能问题
**挑战**: 每个句子都调用GraphRAG,可能很慢

**解决方案**:
1. **批量检索**: 一次性检索整篇文档的所有实体,缓存结果
2. **并发处理**: 使用asyncio并发查询多个句子
3. **快速过滤**: 先用规则过滤明显无幻觉的句子(如问候语)

### 7.2 证据不足问题
**挑战**: GraphRAG可能检索不到相关证据

**解决方案**:
1. **多策略查询**: 结合local/global/drift三种方法
2. **降级处理**: 证据不足时标记为E3,但仍由LLM基于常识判断
3. **人工复核**: 对E3和E4的结果要求人工复审

### 7.3 提示词注入攻击
**挑战**: 恶意文本可能干扰LLM判断

**解决方案**:
1. **输入清洗**: 转义特殊字符
2. **提示词隔离**: 使用明确的分隔符
3. **后处理验证**: 检查输出JSON的合法性

1. **证据驱动**: 不是盲目依赖LLM,而是基于结构化知识做判断

3. **可解释性**: 输出推理过程和证据等级
