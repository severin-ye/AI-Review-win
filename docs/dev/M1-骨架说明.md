# M1 工程骨架说明

> 完成日期：2026-07-17。设计文档：`docs/design/AI-Review-Electron-重构设计.md`；调研依据：`docs/research/调研笔记.md`。

## 架构

Electron 主进程（Node）启动时拉起 Python FastAPI sidecar（动态空闲端口），轮询 `/api/health` 就绪后才创建窗口；退出时 `taskkill /T` 清理进程树。渲染层（React SPA）通过 `window.api.getBackendUrl()` 拿后端地址，业务全部走 HTTP；进度推送走 SSE。

## 常用命令

```bash
npm.cmd install            # 安装前端依赖（.npmrc 已配 Electron 镜像）
npm.cmd run backend:setup  # 创建 backend/.venv 并装后端依赖
npm.cmd run dev            # 启动 electron-vite dev（主进程自动拉起 sidecar）
npm.cmd run build          # 三端构建到 out/
npm.cmd run backend:dev    # 单独调试后端（端口 8765，热重载）
```

注意：本机 Git Bash 中 npm 命令为 `npm.cmd`（node v24.15.0，Kimi 内置运行时）。

## 已知事项

- `.npmrc` 的 `ELECTRON_MIRROR` 会被 npm 警告 "Unknown project config"，但 electron 安装器读取该环境变量，保留无害。
- renderer 产物 ~725kB（react-router+query），M5 打包时可做 manualChunks 优化。
- 后端重依赖（wtpsplit/torch、lancedb、langchain、bge-m3）按 M2/M3 里程碑再加。
- UI 为占位骨架（Tailwind 样式），shadcn/ui 组件在 M4 工作台时引入。
