# M2 解析与分割说明

> 完成日期：2026-07-17。范围：设计文档 5.2 节 ① ingest（docx 解析，表格占位）+ ② segment（SaT 句子分割分块入库），并接通前后端。上游文档：`docs/design/AI-Review-Electron-重构设计.md`、`docs/research/调研笔记.md`、`docs/dev/M1-骨架说明.md`。

## 模块说明

```
backend/app/
  core/user_settings.py   # settings 表读取（docx.has_review_table / segment.min_sentence_length，带点分键兼容）
  pipeline/
    __init__.py
    common.py             # project_dir()、set_document_status()
    ingest.py             # ① docx → tables.docx / no_table.docx / parsed.md（移植旧 table_utils 思路）
    segment.py            # ② SentenceSplitter（SaT 单例 + 正则回退）、短句合并、章节/参考文献分块入库
  api/documents.py        # 上传 / 列表 / run / detail / parsed / DELETE（替换 M1 stub）

  api/client.ts           # uploadDocument/listDocuments/runDocument/getDocumentDetail/getParsed/deleteDocument
  pages/DocumentsPage.tsx # 上传、状态表格、解析按钮、行展开 blocks 树 + parsed.md 查看
backend/tests/test_pipeline.py  # 全链路 TestClient 测试 + 分句/合并单测
```

## 数据流

```
original.docx（上传落盘 projects/<doc_id>/）
  → [ingest] 可选移除首个表格（docx.has_review_table，默认 Y）
  → 其余表格按序抽取 → tables.docx；原位替换占位符 [{表格不予审校_N}] → no_table.docx
  → no_table.docx → parsed.md（Heading 1-3 / 标题 1-3 → #/##/###，空行保留）
  → 状态 uploaded → parsed
  → [segment] parsed.md 逐行：
      # 标题行 / 「一、」「（一）」正则 → chapter 边界（标题保留为块内首句）
      [{表格不予审校_N}] 占位符行 → 独立成句，不参与分句与短句合并
      连续 [N]: / [N]：行 → 参考文献块 is_reference=True（默认不审校）
      其余 → SaT sat-3l-sm 分句 → <10 字碎片合并入前一句（不丢文本）
      单块 >1000 字在句子边界拆分
  → blocks / sentences 入库（重跑先清旧数据及下游 queries/evidence/corrections）
  → 状态 parsed → segmented（任一步失败：documents.error 记录 + failed，job/job_events 记录开始/结束/错误）
```

API：`POST /api/documents`（multipart .docx）、`GET /api/documents`、`POST /api/documents/{id}/run`（同步 ingest+segment，可重跑）、`GET /api/documents/{id}/detail`（blocks+sentences）、`GET /api/documents/{id}/parsed`（纯文本）、`DELETE /api/documents/{id}`（库行+项目目录）。

## SaT 模型状态：**已下载并实模型验证通过** ✅

- 本机 huggingface_hub 0.36 与 hf-mirror 的 HEAD 元数据校验不兼容（`FileMetadataError: Distant resource does not seem to be on huggingface.co`，镜像在 308→302 重定向链中丢失 `X-Repo-Commit`），`SaT("sat-3l-sm")` 直连与 `HF_ENDPOINT=https://hf-mirror.com` 均失败。
- 解决：用 curl 从 hf-mirror 手动下载到本地目录（config.json 965B + model.safetensors 854,658,420B ≈815MB，以及 facebookAI/xlm-roberta-base 的 tokenizer 三件套），落在 `backend/.data/models/{sat-3l-sm,xlm-roberta-base}/`。
- `SentenceSplitter` 新增环境变量支持：`AI_REVIEW_SAT_MODEL_PATH`（本地模型目录）+ `AI_REVIEW_SAT_TOKENIZER_PATH`（本地 tokenizer 目录）；未设置时仍走 `SaT("sat-3l-sm")` HF 自动下载。
- 实模型验证输出（`backend = sat`，中文医学文本 5 句）：

```
1. 患者男性，56岁。
2. 因反复胸闷、气短3个月入院。
3. 查体示双肺呼吸音清，未闻及干湿性啰音。
4. 血常规提示白细胞计数轻度升高。
5. 予以抗感染、化痰及对症支持治疗后症状好转，复查胸部CT未见明显异常。
```

- 正则回退路径已被 pytest 覆盖（`AI_REVIEW_SEGMENTER=rule`，含数字小数点不拆、短句双向合并用例）。

## 已知问题 / 后续注意

1. **Electron 启动的后端默认读不到本地模型环境变量**：sidecar 不带 `AI_REVIEW_SAT_MODEL_PATH` 时会尝试 HF 下载，本机网络下会回退正则。M3 模型管理（设置页预下载/状态展示，`GET /api/models/status` 设计文档已列）应把本地目录方案正式化。
2. **章节正则误判风险**：正文以「一、」「（一）」开头且 ≤60 字的行会被当作章节标题（如"一、二线城市…"）。属可接受误判，后续可在审校工作台人工调整。
3. **占位符句子会进入 block 文本**：M3 retrieve/review 阶段需按 `^\[\{表格不予审校_\d+\}\]$` 跳过这些句子（导出时按占位符还原表格）。
4. **SaT 每段独立分句**：跨段落的语义连句不会合并（段落即审校最小上下文，符合预期）。
5. `blocks.is_reference` 为 M2 新增列，`init_db` 内含轻量 ALTER TABLE 迁移，旧 dev 库无需删除。
