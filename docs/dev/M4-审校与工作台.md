## 流程

```
文档（segmented / retrieved / pending_manual / manual_done 均可重跑）
  → POST /api/documents/{id}/review（LLM 配置预检：缺失 → 400，数据与状态均不变）
  → 后台线程 + job（type=review），SSE 事件：start / progress(每 block 一条) / warning / done / error
  → 清理旧 corrections：默认只清 decision=pending（已人工决定的保留，对应句子本次跳过）；force=true 全清
  → 状态 → reviewing
  → 逐 block（按 idx 序）：
      跳过：is_reference 块（除非 segment.review_references=true）、纯表格占位符 block、
            占位符句（不参与 [S] 编号）、已决定句（重跑默认模式）
      system = settings review.prompt（默认内置医学审校模板）
      user   = 块原文（[S1][S2]…）+ 每句证据（[E1][E2]…，含 向量/关键词·来源标签；逐句无证据注明）
      chat_json（M3 客户端，json_object 优先 + 降级提取）→ corrections
      校验：sentence_id 在块内、original/suggestion 非空 → 非法条目丢弃 + job_events warning；
            error_type/severity 越界收敛默认值；evidence_ids 按该句 [E] 编号映射回 evidence 行 id
      corrections 入库（decision=pending）
  → 状态 reviewing → pending_manual；无 pending（零建议 / 重跑后全已决定）→ manual_done
  → LLM 调用中途失败：状态 → failed（error 落库），job error

人工决定（每条即写库，可随时关闭续作）：
  POST /api/corrections/{id}/decision {decision: accepted|rejected|custom|pending, custom_text?}
    accepted=采纳修订(2) / rejected=保留原文(1) / custom=自定义(3，必须 custom_text) / pending=撤销
    → 写 decided_at；全部决定完成（无 pending）→ 文档 manual_done；撤销 → 回 pending_manual
  POST /api/documents/{id}/decisions/batch {filter:{severity?,error_type?,decision?=pending}, action:accept|reject}
    → 批量处理，返回 {affected, document_status}
```

## prompt 设计

**system**（`review.prompt`，默认值见 `user_settings.DEFAULT_REVIEW_PROMPT`，设置页可编辑）要点：

- 角色：资深医学文稿审校专家，逐句核查；
- 四类问题：事实错误 / 术语错误 / 语法错误 / 格式错误；
- 事实性判断若提供了检索证据，**必须在 explanation 中引用证据编号**（如 [E1]）；
- `original` 必须逐字摘录句中被修改片段（供程序定位替换，M5 导出依赖此约束）；
- severity 语义：high=事实性错误（可能误导诊疗）、medium=术语/明显语法、low=格式措辞；
- 无问题的块返回空 corrections 数组；禁止无依据风格改写。

**user** 结构（`build_user_prompt()`）：

```
请审校以下文稿块：

【正文】
[S1] 句子一……
[S2] 句子二……

【检索证据】
[S1] 的证据：
[E1]（向量 · 来源《高血压指南.pdf》）证据文本……
[E2]（关键词 · 来源《说明书.pdf》）……
[S2]：无检索证据。

请逐句检查并输出 corrections（无问题的块返回空数组）。
```

- **兼容两种前置状态**：已 retrieve 的句子带 3+3 证据；未 retrieve / 检索关闭（retrieve.enabled=false）时逐句注明「无检索证据」，全块无证据再附加「基于医学常识审慎判断、explanation 注明无证据支持」提示——同一 prompt 模板覆盖两种场景。
- schema_hint（`{"corrections":[{sentence_id, original, suggestion, error_type, severity, evidence_ids, explanation}]}`）附加在 user 末尾引导结构化输出。
- 证据编号按句内 score 降序（同分按 id）编排，与 detail API / 前端证据面板的 E 序号一一对应，corrections.evidence_ids 落库时映射回 evidence 行 id（可追溯）。

## UI 交互说明（审校工作台）

路由：`/workbench/:docId`（文档库「继续审校 / 查看审校」或 AI 审校完成自动跳转）。

**布局**（四区）：

- **顶栏**：返回、文档名 + 状态、决定进度条（x/y）、三个筛选器（决定状态 / 错误类型 / 严重度）、批量操作（「接受全部 low」直接执行；「拒绝全部」弹确认 Dialog，仅作用待决定项，可逐条撤销）
- **左栏·文档原文**：按 block 渲染；有待决定 correction 的句子按最高 severity 着色（high 红 / medium 橙 / low 黄），全部已决定的句子淡化；点击高亮句 → 中栏定位到对应卡片（蓝圈标记）
- **中栏·correction 卡片**：类型/严重度徽章 + 块号章节；原句视图（original 片段定位回整句，删除字符红色删除线）vs 建议视图（插入字符绿色高亮，字符级 LCS diff）；explanation；操作按钮；已决定卡片灰绿标记 + 决定徽章（已采纳/已保留原文/已自定义），custom 卡显示 custom_text；决定后自动前进到下一张待决定卡片
- **右栏·证据面板**：选中卡片所属句的全部证据（E 序号 + vector/keyword 标签 + 来源文档 + chunk 全文）；AI 在 explanation 中引用的证据（evidence_ids）蓝框 + 「AI 引用」标记；无证据句给出原因说明

**决定流**：按钮/快捷键 → zustand 乐观更新（卡片即时变色、进度条即时前进）→ API 写库；失败回滚 + toast.error 提示；批量操作成功后重载文档。

**快捷键表**：

| 按键 | 作用 |
|---|---|
| `j` / `↓` | 下一张卡片 |
| `k` / `↑` | 上一张卡片 |
| `1` | 保留原文（rejected） |
| `2` | 采纳（accepted） |
| `3` | 自定义（展开输入框，默认填入建议文本） |
| `Enter` | 确认自定义（输入框内；Shift+Enter 换行） |
| `Esc` | 取消自定义输入 / 关闭弹窗 |

快捷键在自定义输入框聚焦时自动失效（按键交给输入框）；1/2/3 仅对待决定卡片生效。

**shadcn/ui 引入方式**：手工创建 `renderer/src/components/ui/`（button/badge/card/separator/dialog/sonner）+ `lib/utils.ts` 的 `cn()`（clsx + tailwind-merge，新装 tailwind-merge 依赖）；不跑 shadcn CLI（交互式）。dialog/sonner 为无 radix 的最小实现（遮罩+Esc 关闭；命令式 toast()/toast.success()/toast.error()）。

**文档库页**：状态 segmented/retrieved 显示「AI 审校」（触发 review + SSE 进度横幅，完成自动跳工作台；未配置 LLM 时后端 400 友好提示）；pending_manual 显示「继续审校」；manual_done 显示「查看审校」；新增 reviewing/pending_manual/manual_done 状态徽章。

## 已知问题

1. **决定并发**：工作台乐观更新以最后操作为准，多人/多窗同时决定同一文档无冲突检测（单机桌面应用场景下可接受）。
2. **batch 后重载**：批量操作直接重载整个 detail（数据量小；若后续文档超千条 correction 可改局部更新）。
3. **original 定位**：LLM 若未逐字摘录 original（违背 prompt 约束），卡片降级为片段级 diff 展示（不回整句）；M5 导出替换时同样需兜底（计替换失败记日志）。
4. **review 期间文档状态**：后台审校中文档为 reviewing，此时重复触发 review 会被 400 拒绝（REVIEWABLE_STATUSES 不含 reviewing），前端按钮也已禁用。
5. **自定义文本与句子的关系**：custom_text 仅落库，卡片「建议」视图直接整体展示 custom_text（不做与句子的 diff），导出语义（替换 original → custom_text）留待 M5。
6. 并行代理负责的 `backend/.data/models/` 模型下载与本里程碑无交集；其 `embeddings.py` 兼容改动已共存验证（全量测试通过）。
