## 导出语义（§5.2⑥）

**前置**：状态 `pending_manual` / `manual_done` / `done`（其他状态 400）；没有任何 accepted/custom 决定时仍导出（等同原文），`adopted=0`。

**双版本**（输出到 settings `output.dir`，默认 `projects/<doc_id>/exports/`；文件名沿用旧版）：

| 版本 | 文件 | accepted | custom | rejected / pending |
|---|---|---|---|---|
| 清洁版 | `{原名}_审校修订1_.docx` | 替换为 suggestion | 替换为 custom_text | 保留原文 |
| 留痕版 | `{原名}_审校修订2_.docx` | `｛～原文:{original} AI:{suggestion}～｝` | `｛～原文:{original} 用户:{custom_text}～｝` | 保留原文 |

**替换锚点**：correction.original 在其所属**句子文本内**做一次替换（`blocks.text` 由句子经 `\n` 连接而成，句级替换与"以句子为锚点在块内替换一次"等价，且避免同文重复句误伤）。锚点找不到时降级：记 job_events `warning` 事件 + 保留原文（M4 已知问题——LLM 未逐字摘录 original 的场景）；warnings 列表去重后随响应与 exports_json 返回。

**段落结构还原**：block → 段落（块内句子直接拼接还原连续文本）；chapter 标题行（块内首句且与 chapter 同文）用 Heading 样式；占位符句 `[{表格不予审校_N}]` 独立成段；is_reference 块原文照搬（不审校）。随后：

1. **表格还原**：占位符段落 → tables.docx 中第 N 个表格（deepcopy + addprevious 原位插入；旧版按段落序号 insert 在已含表格的 body 中可能错位，此处修正）；
2. **首行缩进**：全文段落 `first_line_indent = Inches(docx.first_line_indent)`（默认 0.5，表格单元格段落不受影响，与旧版一致）。

**落库**：状态 → `done`；`documents.exports_json` = `{clean, marked, adopted, warnings, exported_at}`；job（type=export）事件 `start / progress(每版本) / warning / done / error`。导出失败**不破坏审校成果**：状态保持原状仅记 error（与 ingest/segment 的 failed 语义不同）。`adopted` = accepted + custom 决定数（锚点缺失降级不影响决定计数，见 warnings）。

## 模型管理（修复 M2/M3 遗留的 sidecar 模型路径问题）

- **自动探测**（`core/config.py`）：`AI_REVIEW_SAT_MODEL_PATH` / `AI_REVIEW_SAT_TOKENIZER_PATH` / `AI_REVIEW_EMBEDDING_MODEL_PATH` 环境变量优先；未设置时 `<data_dir>/models/{sat-3l-sm,xlm-roberta-base,bge-m3}` 存在即用。Electron sidecar 因此无需注入任何环境变量。
- `GET /api/models/status`：sat / sat_tokenizer / bge_m3 三项，各含 `path / exists / size_bytes / file_count / missing_files / ready / loaded`（loaded 只读探测单例状态，不触发加载）。
- `POST /api/models/download`：后台线程 curl `hf-mirror.com` 逐文件下载（`-L --retry 3 -C -` 断点续传，与 M2/M3 手工命令一致）；已存在且非空的文件跳过；失败文件清残留 + warning 事件；同时间仅允许一个下载任务（409）。下载清单：
  - `sat`：segment-any-text/sat-3l-sm → config.json, model.safetensors
  - `sat-tokenizer`：FacebookAI/xlm-roberta-base → tokenizer.json, tokenizer_config.json, sentencepiece.bpe.model
  - `bge-m3`：BAAI/bge-m3 → 10 文件（含 1_Pooling/config.json；该仓库只有 pytorch_model.bin，无 model.safetensors，见 M3 勘误）
- 下载完成后新进程/下次加载自动生效；**已加载的旧单例不热切换**（重启后端生效）。

## api_key 安全（务实方案）

- GET /api/settings：`llm.api_key` 只返回掩码（`****` + 后 4 位），空值原样返回。
- PUT /api/settings：`llm.api_key` 值以 `****` 开头 → 跳过写入（设置页整包回传掩码时保留原值）；空串 = 清除；其他值 = 覆盖。
- OS 钥匙串（keytar / keyring）留作后续工作（设计文档 §8）；settings 表目前仍为明文存储，仅 API 层不回读。

## 打包步骤（用户侧操作手册）

```bash
# 1. 后端 → 单文件 exe（venv 内需 pip install pyinstaller）
npm run dist:backend        # = cd backend && .venv\Scripts\python -m PyInstaller ai-review-backend.spec --noconfirm
                            # 产物 backend/dist/ai-review-backend.exe（约 361MB，含 torch/transformers CPU 版）
# 2. 前端构建 + NSIS 安装包
npm run dist                # = npm run build && electron-builder（配置 electron-builder.yml）
                            # 产物 release/ 下 NSIS 安装包
```

- spec：onefile、console 保留（sidecar 日志透传）、`collect_all(lancedb/langchain/langchain_community/wtpsplit/jieba)` + `pkg_resources/jaraco.*` hidden imports、排除 pytest/tkinter/nvidia/cudnn、模型文件不入包。
- electron-builder.yml：`extraResources` 把 `backend/dist/ai-review-backend.exe` 映射到 `resources/backend/`；sidecar prod 模式 `spawn resourcesPath/backend/ai-review-backend.exe --port <port>`（`AI_REVIEW_PACKAGED=1` → 数据目录 `%APPDATA%/ai-review/`）；NSIS 非一键安装、可改目录、桌面/开始菜单快捷方式。
- **模型不打包**：首次安装后经 设置页 → 模型管理 从 hf-mirror 预下载（断点续传），`<数据目录>/models/` 自动探测生效。

## 已知问题 / 后续注意

1. **PyInstaller exe 未验证**：`pyi_rth_pkgres` 运行时钩子 `import pkg_resources` → `from jaraco.text import ...` 失败——venv 的 jaraco 实际位于 `setuptools/_vendor/jaraco/`（setuptools 71+ vendored 命名空间），PyInstaller 不复制该命名空间映射。候选修法（下次验证）：spec 增加 `collect_all("setuptools._vendor")` 或用运行时 hook 把 `setuptools._vendor` 注入 `sys.path` / `jaraco` 命名空间；或 `excludes` 移除 `pkg_resources` 使该钩子不触发（需确认 lancedb/pyarrow 不依赖 pkg_resources）。按里程碑约定：脚本+文档保留，不算失败。
2. **标题层级扁平化**：导出时 chapter 标题统一 Heading 1（原 ##/### 层级在 segment 阶段未入库）；如需还原层级，blocks 需加 level 列（后续里程碑）。
3. **块级段落合并**：segment 把同章多个自然段并入同 block（上限 1000 字），导出按 block 成段，原段落边界不完全还原（文本无损，仅分段粒度）；占位符与标题边界始终正确。
4. **模型热切换**：模型下载完成后已加载的 SentenceSplitter/EmbeddingProvider 单例不自动切换，重启后端生效（单机桌面场景可接受）。
5. **api_key 仍明文存 settings 表**：仅 API 层不回读；OS 钥匙串（keytar/keyring）留作后续工作。
6. **download 进度粒度**：curl 以文件为单位发 progress 事件（大文件如 pytorch_model.bin 2.27GB 期间无字节级进度）；如需字节级进度可改解析 curl `--progress-bar` 输出或换 httpx 流式下载。
7. **机器进程清理说明**：本次验证曾发现 8793/8794（M2/M3 历史遗留 uvicorn）与 8795（未知历史 python）端口占用，已一并 taskkill 清理；M5 自身启动的全部进程（uvicorn 8797、vite 8795、PyInstaller exe 冒烟实例）均已关闭并复核端口释放。
